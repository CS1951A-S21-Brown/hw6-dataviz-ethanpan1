    1. D3 allowed me to easily apply the same transform function to every item (bar, dot, label) that I needed to transform.
    2. D3 easily handled color through interpolating a gradient or providing a scheme set, improving readability and visual interest.
    3. D3, through SVGs, allowed for easily scalable graphics, making my visualizations less reliant on hard-coded positioning and allowing for legible rescaling.

    1. D3 would not be useful for creating a really simple visualization that doesn't need to be rescaled; other libraries like matplotlib can do that more easily.
    2. D3 may not be the best tool if the final visualization is not intended to be displayed on a web browser.
    3. Since D3 is written in JS, it fits less well in a streamlined Python-based pipeline that relies on pandas or numpy analysis.

  4. I think my dashboard has moderate accessibility. It follows color contrast guidelines fairly well, as described by Miller's article. For the most part, data points are labelled directly, although there is a separated color legend for the third graph. For the most part, color-blind users would be able to use my dashboard well.

    For assistive technology users, my dashboard is less accessible. Also to Miller's article, I refrain from complex tooling, but I do use hover overlays for the second two graphs. In addition, I don't use alt text, and the third graph is perhaps too busy to be comprehensible for AT. In the future, I would remove the hover overlays, turn the third graph into multiple graphs, and add more alt text.

    Since I don't use symbols or sound, my dashboard is fairly accessible to those with various cognitive disabilities or those that are Deaf (according to Schepers's article).

  5. For certain places that I did not choose the more accessible option (e.g., the color legend), the main reason was because I didn't know how to make the more accessible option through D3. For instance, while I could've added hard-coded direct color labelling for one graph, I didn't know how to do it for all twelve genre graphs altogether. Time was also a major factor. For example, I know how to expand the third graph into a sequence of five graphs with one publisher on each, but I didn't have the time to take this extra step.

Extra Credit: I added a dynamic statistical calculation in the third graph. For any given publisher in a given genre, if you mouse over a data point, a linear regression line for that publisher's performance is shown. This calculation is statistical because it approximates the general trend of the publisher's performance via the x-y coordinates of the data points in the graph.
